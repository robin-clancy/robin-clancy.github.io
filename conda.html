Conda

Reference for how to use Conda (specifically Anaconda in this case) to create virtual environments and configure them to work with Jupyter notebooks.


This page provides a reference for myself or others for how to generate a virtual environment using Anacconda from scratch. The main use case discussed is generating a new Python evnrionment and configuring it to work with Jupyter notebooks. It is good practice to generate a new python environment for each project.

Step 1. Installing Anaconda
Find a link to the command line installer for Anaconda (Miniconda may be a preferable alternative)
e.g. https://repo.anaconda.com/archive/Anaconda3-2020.07-MacOSX-x86_64.sh.

wget <link_to_installer>
bash installer.sh

Let it initialize conda for you when it asks.
Check .bashrc to see if this puts in the path correctly.
If using a different shell (e.g. tcsh on atmos computers check .tcshrc)
For zsh on Mac use: bash -c "conda init zsh"

Verify conda installation: conda info

Step 2: Creating a new virtual environment
To see available environments: conda env list

To create a new environment use one of these:
conda create -n <envname>
conda create -n <envname> python=3.6(e.g., for a specific python version)
conda create --clone <envname> --name <newenvname>

Step 3: Activating an environment
Environments must be activated to update or use them: conda activate myenv
To stop working with an environment: conda deactivate
To remove an environment entirely: conda env remove -n <envname>

Step 4: Setting up environment to work with Jupyter notebooks
Ensure conda environment is activated first.
Install ipykernel package: conda install ipykernel
Then add the environment (i.e. kernel) to Jupyter: python -m ipykernel install --user --name=<envname>

To remove an environment from Jupyter use: jupyter kernelspec list and jupyter kernelspec uninstall <envname>

Step 5: Adding, updating and removing packages
Ensure conda environment is activated first.
To view all packages in current environment: conda list
To install a package: conda install packagename or pip install packagename
To install a specific package version: conda install packagename=1.11 or for stricter constraints: conda install packagename==1.11 To specify conda forge: conda install --channel conda-forge packagename To update a package: conda update packagename To remove a package: conda remove --name myenv packagename Misc tips. To check python version: python --version To check python path: which -a python (1st entry is path used) To see history of environment: conda list --revisions To revert back to an old version of environment: conda install --revision 2
