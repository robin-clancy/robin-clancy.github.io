Python Style

A reference page with templates for how to write appropriately styled Python code. Vaugely based on PEP8 (and eventually specifically Black?) guidelines. Not exhaustive - there's plenty of stuff that's easily remembered or too granular/specific to include.

This source is referenced heavily.

Later I will add downloadable templates for .py and .ipynb files. Also maybe examples of how to use Black.

"""This module's docstring summary line.

Paragraphs are separated with blank lines.
"""

Imports

import os  # standard library imports first
import sys  # in alphabetical order

import some_third_party_lib  # 3rd party stuff next

import local_stuff

Functions

def function_name(param1: type | type2, param2: type = default_val) -> return_type:
    """Description of the function and its arguments.

    Args:
        param1 (type, type2): The first parameter.
        param2 (type): The second parameter. Defaults to default_val.

    Returns:
    	return_type: Description of the return value.
    """
    return_val = param1 + param2  # Comment
    return return_val
